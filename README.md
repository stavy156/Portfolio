# Running the Project

This project consists of two services running in parallel:

* **Spring Boot Application (Port 8080)** — Generates application metrics and simulates system behavior.
* **FastAPI ML Service (Port 8000)** — Collects metrics from Spring Boot, performs anomaly detection and health analysis, and exposes system insights.

Optionally, you can run a **k6 load test** to generate realistic traffic.

---

## Step 1: Start the Spring Boot Application

Run the main Java application:

```bash
./mvnw spring-boot:run
```

or simply run:

```
DemoApplication.java
```

This starts the marketplace application and monitoring system on:

```
http://localhost:8080
```
| `/swagger-ui.html` | Interactive API documentation |
---

## Step 2: Start the ML Service

Open another terminal: (start following if done once)

```bash
cd ml-service
source .venv/bin/activate
uvicorn app.main:app --reload --port 8000
```

This starts the ML inference service on:

```
http://localhost:8000
```

# How the System Works

## 1. User Requests Enter Spring Boot

```
Request
   ↓
MetricsInterceptor
   ↓
Controllers
   ↓
Services
   ↓
Simulated Dependencies
```

Every request records:

* Response time
* Error rate
* Throughput
* Dependency failures
* System reliability metrics

---

## 2. Metrics Aggregation

Every **5 seconds**, Spring Boot aggregates all collected metrics into a single snapshot:

```
MetricsAggregator
        ↓
MetricsSnapshot
        ↓
GET /api/metrics
```

---

## 3. ML Service Polls Metrics

Every **5 seconds**, the FastAPI service fetches the latest metrics:

```
collector.py
       ↓
GET localhost:8080/api/metrics
       ↓
pipeline.py
```

---

## 4. Feature Extraction

The metrics JSON is converted into a numerical feature vector:

```
features.py
        ↓
31 ML features
```

Examples include:

* Average latency
* Error rate
* Throughput
* P95 latency
* Reliability scores
* Trend slopes
* Dependency failures

---

## 5. Anomaly Detection

The ML pipeline detects abnormal behavior using:

### Z-Score Detection

Detects sudden spikes in individual metrics.

### Isolation Forest

Detects unusual patterns across multiple metrics.

```
anomaly.py
```

---

## 6. Health Scoring

A custom scoring engine calculates an overall system health score:

```
health_scorer.py
```

Output:

* HEALTHY
* WARNING
* CRITICAL

with a score between:

```
0 — 100
```

---

## 7. Root Cause Analysis

The system attempts to determine why the anomaly occurred:

```
root_cause.py
```

Examples:

* High database latency
* Increased API failures
* Rising error trends
* Dependency outages

---

## Final ML Output

The ML service produces a human-readable health report:

```json
{
  "status": "WARNING",
  "healthScore": 72,
  "reason": "Database latency increased significantly",
  "prediction": "System performance may degrade further",
  "confidence": 0.87,
  "anomalies": ["dbLatency", "errorRate"],
  "topContributors": [
    "Database latency",
    "API error rate"
  ]
}
```

View the latest analysis at:

```
http://localhost:8000/latest
```
